## Данные

Данные, использующиеся в этом домашнем задании, взяты c  [Kaggle](https://www.kaggle.com/datasets/maksimpotorochin/movie-plots-from-wikipedia-in-russian) – это датасет с описаниями фильмов, сериалов, аниме, мультсериалов и т.д. из русскоязычной Википедии. Всего в этом датасете $44105$ уникальных значения, но в рамках задания я взяла лишь $1500$ случайных строк из датасета.

Почему именно этот датасет?

1) Я люблю кино.

2) Мне показалось, что на этих данных можно построить осмысленный поиск: по каким-то ключевым словам в запросе мы ищем сюжет какого-то кино.

3) В рамках проекта можно будет попробовать построить осмысленную рекомендательную систему, потому что результат поиска = кино и т.д.

Для работы с поиском датасет можно скачать либо с [Kaggle](https://www.kaggle.com/datasets/maksimpotorochin/movie-plots-from-wikipedia-in-russian), либо с [диска](https://drive.google.com/file/d/1VYZJZiouye9M3fy9lpts116PZ4ji_cTU/view?usp=sharing), или разархивировать `films_data.zip`, который лежит в этом репозиттории.

## Структура репозитория

Репозиторий обладает следующей структурой:

```
.
├── README.md
├── films_data.zip
├── index.py
├── main.py
├── preprocess.py
└── requirements.txt
```

Внутри архива `films_data.zip` лежит датасет, из которого берутся данные. Они обрабатываются внутри `preprocess.py`.

Поиск с двумя видами индексов (частотный и BM-25) реализован внутри `index.py`. В файле представлено несколько реализаций: при помощи готовых модулей, при помощи словарей и при помощи матриц. Каждая из этих реализаций может быть использована при поиске.

Сам поиск можно запустить при помощи `main.py`, о чем написано в следующей разделе.

Все необходимые библиотеки и их версии перечислены в `requirements.txt`.

## Запуск кода

Версии, использованные в этом задании: `Python 3.11.9` и `pip 24.2`.

Перед тем как работать с поиском, необходимо

1) Cоздать виртуальное окружение 

```
<!-- для mac os x -->
cd /path/to/informationRetrieval-hw1
python3 -m venv venv
source venv/bin/activate

<!-- для windows -->
cd C:\path\to\to/informationRetrieval-hw1
python -m venv venv
venv\Scripts\activate.bat
```

2) Установить необходимые библиотеки

```
pip install -r requirements.txt
```

3) Разархивировать `films_data.zip`, чтобы достать из него файл с расширением `.csv`. Оригинальный файл слишком большой для загрузки в репозиторий, поэтому был создан архив. Файл так же можно скачать либо с [Kaggle](https://www.kaggle.com/datasets/maksimpotorochin/movie-plots-from-wikipedia-in-russian), либо с [диска](https://drive.google.com/file/d/1VYZJZiouye9M3fy9lpts116PZ4ji_cTU/view?usp=sharing).

Для работы с поиском необходимо запустить только `main.py` (остальные файлы являются вспомогательными). После его запуска данные из `films_data.csv` будут прочитаны, предобработаны и векторизованы / записаны в словарь. На данном этапе данные никак не кешируются и процесс обработки повторяется при каждом запуске кода (время работы зависит от количества выбранных записей из датасета; для $1500$ документов у меня предобработка занимала $7-10$ минут).

Когда обработка данных будет закончена, пользователь увидит об этом уведомление и получит инструкцию по работе с поиском.

## Работа с поиском

Можно выбрать тип индекса, который будет использован при поиске: `по частотам` и `BM-25`.

Запрос и тип индекса должны быть разделены `;`. Пример запроса: `ужасы триллер аниме;по частотам`.

Для того чтобы продублировать инструкцию необходимо напечатать `Инструкция`.

Для того чтобы завершить поиск необходимо напечатать `Завершить поиск`.

## Дополнительно

Можно выбрать реализацию поиска. Для этого нужно раскомментировать нужную строчку среди строчек $5-7$ внутри `main.py`. 

```
srch = SearchDefault() # готовые модули
srch = SearchDict() # поиск при помощи словарей
srch = SearchMatrix() # поиск при помощи матриц
```

Можно изменить размер корпуса. Для этого в `preprocess.py` на строчке $15$ нужно поменять значение переменной `N`. Сейчас дефолтное выбранное значение для нее – $1500$ записей из файла `films_data.csv`.

Можно изменить, сколько релеватных документов будет получено на выходе. Для этого в `main.py` на строчке $28$ нужно поменять значение переменной `top`.
